## 3. 关键设计决策

### 3.1 事件总线（EventBus）
所有跨模块通信通过 `core/event_bus.py` 的 `pyqtSignal` 进行，避免模块间直接依赖：

```python
# 典型信号
screening_completed        = pyqtSignal(list)   # 选股完成 → 结果页 + 信号页 + 仪表盘
portfolio_updated          = pyqtSignal()       # 持仓变更 → 仪表盘 + 目标页
rdagent_log                = pyqtSignal(str)    # 因子发现日志 → 因子页
rdagent_completed          = pyqtSignal(list)   # 因子发现完成
rdagent_factors_injected   = pyqtSignal(list)   # 因子注入完成 → 选股页面板刷新（list[dict]）
backtest_ticker_hint       = pyqtSignal(str)    # 选股结果页 → 回测页预填股票代码
```

优点：页面间解耦，新增页面不需修改已有页面代码。

### 3.2 Worker 模式
所有耗时操作（API 调用、模型推理、Docker 操作）均在 `QRunnable` 子线程中执行：

```
UI 线程（Qt 主线程）
  └─ QThreadPool.globalInstance().start(worker)
       └─ worker.run()  ← 子线程
            └─ signals.result.emit(data)  → UI 线程更新
```

信号（`QObject` 子类）与 `QRunnable` 分离定义，避免多重继承问题：
```python
class ScreeningSignals(QObject):
    progress  = pyqtSignal(int, str)
    completed = pyqtSignal(list)
    failed    = pyqtSignal(str)

class ScreeningWorker(QRunnable):
    def __init__(self):
        self.signals = ScreeningSignals()
```

### 3.3 Qlib 降级策略
| 场景 | 正常模式 | 降级模式 |
|------|---------|---------|
| 选股股票池 | `D.list_instruments("us")` | 硬编码 100 支大盘股 |
| 模型预测 | Qlib LGBModel / LSTM | 因子均值简单评分 |
| 回测 | Qlib TopkDropoutStrategy | 等权价格简化回测 |

### 3.4 OpenBB 多源降级
```python
PROVIDER_PRIORITY = ["alpha_vantage", "yfinance"]

for provider in PROVIDER_PRIORITY:
    try:
        result = obb.equity.price.historical(..., provider=provider)
        if result and result.results:
            return result.to_dataframe()
    except Exception:
        continue
return None  # 全部失败 → 调用方显示「暂无数据」
```

### 3.5 Python 3.9 兼容性
每个文件开头（docstring 之后、第一行 import 之前）添加：
```python
from __future__ import annotations
```
解决：
- `X | None` 联合类型语法（3.10+ 才原生支持）
- `dict[str, ...]` / `list[...]` 内置泛型（3.9+ 需要此 future import）

### 3.6 数据存储
| 数据 | 存储位置 | 格式 |
|------|---------|------|
| 持仓记录 | `~/.quantbyqlib/portfolio.db` | SQLite |
| 盈利目标 | `~/.quantbyqlib/portfolio.db` | SQLite（同库不同表）|
| Qlib 市场数据 | `~/.qlib/qlib_data/` | Qlib bin 格式（宿主机，同时挂载进容器）|
| RD-Agent 因子会话 | `~/.quantbyqlib/rdagent_sessions.json` | JSON，含完整因子字段（name/description/ic_mean 等）|
| 已验证因子库 | `~/.quantbyqlib/valid_factors.json` | JSON v2，含 factors 列表（expression/name/description）|
| RD-Agent 工作目录 | `~/.quantbyqlib/rdagent_workspace/` | 宿主机挂载进容器（读写）|
| 日志 | `~/.quantbyqlib/logs/` | loguru 滚动日志 |
| API Key | `项目根目录/.env` | dotenv 格式 |

## 5. 数据流示意

### 量化选股完整流程
```
用户点击「运行选股」
    │
    ▼
ScreeningWorker.run()  ← QThreadPool 子线程
    │
    ├─ StockScreener._get_universe()            获取股票池
    ├─ load_valid_factors()                     加载自定义因子（~/.quantbyqlib/valid_factors.json）
    ├─ strategy.run(universe, topk)             Qlib 模型预测（LightGBM 时注入自定义因子列）
    ├─ _batch_get_changes(tickers)              OpenBB 今日涨跌
    └─ signals.completed.emit(results)
         │
         ├─→ EventBus.screening_completed.emit(results)
         │       ├─→ ResultsPage._on_screening_done()    更新结果表格
         │       ├─→ SignalsPage._on_screening_done()    生成信号列表
         │       └─→ DashboardPage._on_screening_done()  更新仪表盘预览
         │
         └─→ ResultsPage 左侧选中行
                 └─→ StockDetailPanel.load(ticker)
                         ├─ AnalysisWorker（子线程）
                         │   ├─ alpha_reader.get_technical_signal()
                         │   ├─ price_chart.get_chart_data()
                         │   ├─ fundamental.analyze()
                         │   └─ sentiment.analyze()
                         └─ signals.result.emit(report) → 面板渲染
```

### 因子注入完整流程
```
用户在因子发现页点击「✅ 注入选股策略」
    │
    ▼
FactorInjectWorker.run()  ← QThreadPool 子线程
    │
    ├─ _qlib_init_check()                       检查 Qlib 是否就绪
    ├─ session_manager.get_latest()             读取最新 RD-Agent 会话因子
    ├─ _precheck_expression(expr)               语法预检（Max/Min 窗口、嵌套等）
    ├─ get_valid_factors(min_ic=0.03)           合并历史库 + 全量重验证
    │      └─ validate_factor(expr, universe)   D.features → 截面 Spearman IC
    ├─ save_valid_factors(valid_exprs)          持久化到 valid_factors.json（含 name/description）
    ├─ model_cache.clear_cache()                清除旧模型缓存（使下次选股重新训练）
    └─ signals.completed.emit(valid_exprs)
         │
         ├─→ FactorPage._on_inject_completed()  更新因子页注入状态 + 因子标签列表
         └─→ EventBus.rdagent_factors_injected.emit(valid_exprs)
                 └─→ ScreeningPage._build_inj_tags()  量化选股页面板自动刷新
```
