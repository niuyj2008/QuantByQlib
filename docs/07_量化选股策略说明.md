# 量化选股策略说明

> 本文档说明系统内置的5种 Qlib 量化选股策略的实现原理、技术来源与适用场景。

---

## 策略总览

| 策略 | 模型 | 因子集 | 训练窗口 | 目标选出 | 风险类型 |
|------|------|--------|---------|---------|---------|
| 深度学习集成 | LSTM | Alpha158（158个） | 2年（504天） | Top 50 | 平衡型 |
| 短线获利 | GRU | Alpha158（158个） | 6个月（126天） | Top 30 | 进取型 |
| 成长股选股 | LightGBM | Alpha158 + 自定义因子 | 2年（504天） | Top 50 | 稳健型 |
| 市场自适应 | LightGBM | Alpha158 + 自定义因子 | 2年（504天） | Top 50 | 平衡型 |
| 全市场深度学习 | LSTM | Alpha360（360个） | 2年（504天） | Top 50 | 进取型 |

---

## 策略1：🧠 深度学习集成（LSTM）

### 通俗解释
像人类读 K 线图一样，模型按时间顺序"阅读"过去 2 年的每日价量变化，从中提炼出价量非线性规律，预测未来收益排名。

### 技术来源
- **模型类**：`qlib.contrib.model.pytorch_lstm.LSTM`（微软 Qlib 官方实现）
- **因子集**：**Alpha158** — 158 个技术因子，涵盖动量、反转、波动率、成交量比率等

### 关键参数
```
d_feat      = 158     输入维度（对应 Alpha158）
hidden_size = 64      隐藏层神经元数
num_layers  = 2       LSTM 层数
n_epochs    = 10      最大训练轮数
lr          = 0.001   学习率
batch_size  = 512
train_days  = 504     训练数据窗口（约2年）
MAX_UNIVERSE= 300     最大股票池（防内存溢出）
```

### 适用场景
- 中长期趋势跟踪
- 市场方向明确时效果好
- **不适合**：频繁换仓、短期波段操作

### 局限性
- CPU 训练较慢（约 3-8 分钟）
- Alpha158 是标准因子集，无法注入 RD-Agent 自定义因子

---

## 策略2：⚡ 短线获利（GRU）

### 通俗解释
只看最近半年的行情，专门捕捉短期动量效应。GRU 比 LSTM 更轻量，对短序列反应更灵敏，适合活跃交易者。

### 技术来源
- **模型类**：`qlib.contrib.model.pytorch_gru.GRU`（微软 Qlib 官方实现）
- **因子集**：**Alpha158**（与策略1相同）

### 关键参数
```
d_feat      = 158     输入维度
hidden_size = 64      隐藏层神经元数
num_layers  = 2       GRU 层数
n_epochs    = 10
train_days  = 126     ← 核心差异：只用6个月数据（策略1用2年）
pred_days   = 10      预测窗口10天
MAX_UNIVERSE= 300
topk        = 30      ← 更集中：只选30支（其他策略选50支）
```

### LSTM vs GRU 的本质区别
| 维度 | LSTM | GRU |
|------|------|-----|
| 门控结构 | 输入门 + 遗忘门 + 输出门（3个） | 重置门 + 更新门（2个） |
| 参数量 | 较多 | 较少（约75%） |
| 长期记忆 | 更强 | 较弱 |
| 短序列表现 | 一般 | 更好 |
| 训练速度 | 较慢 | 较快 |

### 适用场景
- 活跃交易、持仓周期1-2周
- 市场有明显短期动量时
- **不适合**：长期持有、震荡市

---

## 策略3：🌱 成长股选股（LightGBM）

### 通俗解释
不使用神经网络，而是成千上万棵"决策树"集体投票。每棵树学习一个规则（如"当20日动量>5%且成交量放大时看多"），所有树的结论汇总给出最终评分。模型可解释性强，回撤控制好。

### 技术来源
- **模型类**：`qlib.contrib.model.gbdt.LGBModel`（Qlib 对 LightGBM 的封装）
- **因子集**：**Alpha158 + RD-Agent 注入的自定义因子**（唯一支持因子注入的神经网络以外的策略）

### 关键参数
```
loss            = "mse"
learning_rate   = 0.0421
colsample_bytree= 0.8879  每棵树随机采样88.79%的特征
subsample       = 0.8789  每次迭代随机采样87.89%的数据
lambda_l1       = 205.70  L1正则化（防过拟合）
lambda_l2       = 580.98  L2正则化
max_depth       = 8       树的最大深度
num_leaves      = 210     最大叶节点数
num_threads     = 4
train_days      = 504
topk            = 50
```

### RD-Agent 因子注入机制
```
训练时：Alpha158（158列）+ 自定义因子（N列）→ 合并 → LightGBM 训练
                                                    ↑
                          ~/.quantbyqlib/valid_factors.json 中的表达式
                          通过 Qlib D.features() 计算后追加
```

### 适用场景
- 稳健中长期持有
- 配合 RD-Agent 因子发现，可持续迭代提升
- **不适合**：短期高频换仓

---

## 策略4：🔄 市场自适应（LightGBM + 牛熊切换）

### 通俗解释
在成长股策略基础上，增加了"感知市场温度"的能力：运行前先看 SPY（标普500ETF）最近60天表现，判断牛熊，然后动态调整模型的"激进程度"（学习率）。

### 技术来源
- **模型类**：`LGBModel`（同成长股策略）
- **牛熊检测**：通过 OpenBB 拉取 SPY 60日价格历史，计算涨跌幅判断

### 牛熊检测逻辑
```python
# 每次运行策略时执行
spy_prices = OpenBB.get_price_history("SPY", days=60)
change = (spy_prices[-1] - spy_prices[0]) / spy_prices[0]

if change > 0:    → 牛市，learning_rate = 0.05（更激进，捕捉上涨动量）
if change < 0:    → 熊市，learning_rate = 0.03（更保守，减少过拟合风险）
else:             → 中性，learning_rate = 0.04
```

### 与成长股策略的区别
| 维度 | 成长股 | 市场自适应 |
|------|--------|-----------|
| 学习率 | 固定 0.0421 | 动态 0.03/0.05 |
| 牛熊感知 | 无 | 有（SPY 60日） |
| 因子注入 | 支持 | 支持 |
| 网络依赖 | 无 | 需要 OpenBB/yfinance 拉 SPY |

### 适用场景
- 跨越牛熊周期的长期使用
- 市场状态变化明显时优势最大
- **注意**：OpenBB 网络不通时回退到固定参数

## 策略5：🌐 全市场深度学习（LSTM + Alpha360）

### 通俗解释
最"重"的策略，用360个因子（是其他策略的2.3倍），覆盖更广的股票池（最多400支）。Alpha360 在 Alpha158 基础上增加了更多高频微结构因子，捕捉被常规策略忽视的信号。

### 技术来源
- **模型类**：`qlib.contrib.model.pytorch_lstm.LSTM`（同策略1）
- **因子集**：**Alpha360** — 360个因子（Alpha158 + 额外的成交量微结构、高频统计因子）

### 关键参数
```
d_feat      = 360     ← 核心差异：360维输入（其他LSTM是158）
hidden_size = 128     ← 更大的隐藏层（策略1是64）
num_layers  = 1       ← 更少的层数（节省内存）
n_epochs    = 8       ← 训练轮数减少（特征多，收敛更快）
MAX_UNIVERSE= 400     ← 更大的股票池（策略1是300）
topk        = 50
```

### Alpha158 vs Alpha360 的区别
| 因子集 | 因子数 | 额外覆盖 | 适合 |
|--------|--------|---------|------|
| Alpha158 | 158 | 标准技术因子 | 标普500蓝筹 |
| Alpha360 | 360 | + 高频微结构、多周期统计 | 全市场含中小盘 |

### 适用场景
- 挖掘市场冷门股、中小盘机会
- 全市场宽基覆盖（NYSE + NASDAQ）
- **注意**：360个特征占用更多内存，CPU 训练更慢（约5-12分钟）

## 共享机制说明

### 24小时训练缓存
所有策略的训练结果都会缓存24小时。缓存键包含：策略名、股票池、数据截止日期、注入因子哈希。相同条件下第二次运行秒出结果。

### Fallback 降级机制
当 Qlib 数据不可用或模型训练失败时，所有策略自动降级为 **yfinance 规则打分**：
```
综合评分 = 平均(
    20日动量评分,   # (close[-1]/close[-20] - 1) × 5 + 0.5
    60日动量评分,   # (close[-1]/close[-60] - 1) × 3 + 0.5
    MA20偏离评分,   # (price/MA20 - 1) × 10 + 0.5
    OBV趋势评分    # OBV变化率 × 2 + 0.5
)
```

### 信号生成规则（百分位排名）
```
Top 10%（≥90百分位）         → BUY  强
Top 30%（≥70百分位）且持仓   → HOLD 中
Top 30%（≥70百分位）未持仓   → WATCH 中
Bottom 20%（<20百分位）且持仓 → SELL 强
其余                         → HOLD 弱
```

### PyTorch 官方 Bug 修复补丁
系统对 Qlib 的 LSTM/GRU 模型打了一个补丁，修复了 `best_param` 未初始化导致的训练崩溃问题，以及特征矩阵中 NaN 值导致梯度爆炸的问题。这是 Qlib 上游的已知缺陷，在本系统中已自动修复。

## 选策略建议

```
刚开始使用        → 成长股选股（最稳健，可解释）
想捕捉短期机会    → 短线获利（持仓1-2周）
追求全自动适应    → 市场自适应（无需手动判断牛熊）
已注入自定义因子  → 成长股选股 或 市场自适应（因子注入生效）
想覆盖更多股票    → 全市场深度学习（但要有耐心等待训练）
```
