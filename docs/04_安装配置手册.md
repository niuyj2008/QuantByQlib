# 04 安装配置手册

## 1. 环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.9+ | 推荐 3.9.6 或 3.11 |
| Docker Desktop | 任意 | 仅因子发现功能需要 |
| macOS | 12+ | Windows 次要支持 |
| 内存 | 8 GB | 运行 DistilBERT 情绪模型需要 |
| 磁盘 | 10 GB | Qlib US 数据约 5 GB，RD-Agent 镜像约 2 GB |

---

## 2. 安装步骤

### 2.1 克隆项目

```bash
git clone https://github.com/yourname/QuantByQlib.git
cd QuantByQlib
```

### 2.2 安装 Python 依赖

```bash
pip3 install -r requirements.txt
```

> **macOS M 系列芯片注意**：lightgbm 需要 OpenMP 支持，若提示 `libomp.dylib` 缺失：
> ```bash
> brew install libomp
> ```

### 2.3 配置 API Key

在项目根目录创建 `.env` 文件（参考 `.env.example`）：

```ini
# OpenBB 数据接口 Key（基本面 / 新闻 / 行情）
FMP_API_KEY=your_fmp_key_here
FINNHUB_API_KEY=your_finnhub_key_here
ALPHA_VANTAGE_API_KEY=your_alpha_vantage_key_here

# AI 分析报告 — 主力 LLM（必填，用于个股 AI 分析报告）
ANTHROPIC_API_KEY=your_anthropic_key_here

# AI 分析报告 — 降级 LLM（可选，Claude 过载时自动切换）
DEEPSEEK_API_KEY=your_deepseek_key_here

# RD-Agent 因子发现（DeepSeek 同一 Key，用途不同）
CHAT_MODEL=deepseek/deepseek-chat

# AI 报告输出目录（可选，留空则使用默认路径）
# 默认：~/Documents/美股交易日记/reports/
# REPORTS_DIR=/custom/path/to/reports
```

> **AI 报告功能说明**：`ANTHROPIC_API_KEY` 用于个股 AI 分析报告（主力）。Claude 服务过载（429/503）时自动降级到 DeepSeek（需同时配置 `DEEPSEEK_API_KEY`）。两者均未配置时报告功能不可用，但不影响其他功能。

> API Key 获取方式见 [docs/05_数据源与API配置.md](05_数据源与API配置.md)

### 2.4 启动应用

```bash
python3 main.py
```

首次启动时：
- Qlib 数据未检测到 → 应用以降级模式运行（选股使用备用股票池，回测使用简化模式）
- 进入「参数配置」页面可下载 Qlib 数据（约 5 GB，需数小时）

## 3. 功能快速入门

### 3.1 查看持仓（无需 Qlib）

1. 点击左侧「💼 持仓管理」
2. 点击「+ 买入」输入交易记录
3. 点击「刷新价格」获取当前市价
4. 自动计算盈亏、回报率

### 3.2 运行量化选股

> 建议先完成 Qlib 数据下载，否则使用备用股票池和简化评分

1. 点击「🔍 量化选股」
2. 选择策略（首次推荐「成长股选股 / LightGBM」，速度最快）
3. 设置 Top-K（默认 20）
4. 点击「▶ 开始选股」，等待进度条完成
5. 自动跳转「📈 选股结果」页

### 3.3 查看个股详情

在选股结果页，点击左侧表格任意一行：
- 右侧面板自动加载四维数据（约 5-10 秒）
- 点击「加入持仓」可直接开仓
- 点击「运行策略」可单独分析该股

### 3.4 查看交易信号

1. 完成量化选股后，点击「⚡ 交易信号」
2. 信号已自动生成（BUY/SELL/HOLD/WATCH）
3. 使用顶部筛选器筛选「买入」信号
4. 点击「📥 导出 CSV」保存信号列表

### 3.5 策略回测

1. 点击「⏱ 策略回测」
2. 配置参数：策略、日期范围（建议 2020-01-01 ~ 2024-12-31）、初始资金
3. 点击「▶ 开始回测」
4. 查看净值曲线（紫色策略 vs 灰色 SPY）和 KPI 指标

### 3.6 设置盈利目标

1. 点击「🎯 盈利目标」
2. 选择风险偏好（稳健/平衡/进取）
3. 点击「+ 新建目标」填写表单
4. 系统自动评估可行性并推荐策略组合

### 3.7 因子发现（RD-Agent）

> 需要 Docker Desktop 运行 + DEEPSEEK_API_KEY + 首次拉取镜像（约 2 GB）

**首次使用**（拉取镜像）：
```bash
docker pull msrarambler/rd-agent:latest
```

1. 点击「🤖 因子发现」
2. 检查 Docker 状态显示「🟢 运行中」
3. 点击「▶ 启动因子发现」
4. 实时查看日志，发现因子后自动显示在右侧表格
5. 点击「📥 导出」保存因子列表

## 4. Qlib 数据下载（推荐）

数据量约 5 GB，下载后量化功能将完整运行。

### 方式一：在应用内下载

1. 「⚙️ 参数配置」→「数据管理」→「下载 US 数据」
2. 等待进度完成（网速约 1-3 小时）

### 方式二：命令行下载

```bash
# 方法 A：Qlib 官方采集器
python3 -m qlib.run.get_data qlib_data --target_dir ~/.qlib/qlib_data/us_data --region us

# 方法 B：从镜像站下载预编译包（更快）
python3 scripts/download_qlib_data.py
```

下载完成后重启应用，Qlib 会自动初始化。

## 5. LLM 报告配置（可选）

AI 分析报告的 LLM 参数可在 `config/app_config.yaml` 中调整：

```yaml
llm:
  model: "claude-opus-4-6"       # 改为 claude-haiku-4-5-20251001 可大幅提速（质量略降）
  max_tokens: 1500               # 报告长度上限，可按需增减
  report_save: true              # false = 不保存 MD 文件，仅在界面展示
  reports_dir: ""                # 留空使用 REPORTS_DIR 环境变量或默认路径
```

报告默认保存路径：`~/Documents/美股交易日记/reports/YYYYMMDD/TICKER_analysis.md`

## 6. 常见问题

### Q: 启动时提示「Qlib 数据未找到」
正常现象，应用以降级模式运行。进入「参数配置」下载数据后重启即可。

### Q: 个股分析显示「暂无数据」
API Key 未配置或已超出免费限额。检查 `.env` 文件中的 Key 是否正确。

### Q: Docker 状态显示「🔴 未连接」
请确认 Docker Desktop 已安装并正在运行（菜单栏有 Docker 图标）。

### Q: 因子发现提示「RD-Agent 镜像未找到」
在终端执行：
```bash
docker pull msrarambler/rd-agent:latest
```
等待约 2 GB 镜像下载完成后重试。

### Q: lightgbm 报错 `Library not loaded: libomp.dylib`
```bash
brew install libomp
```

### Q: 回测结果显示「降级：等权价格模式」
Qlib 数据未初始化，回测使用简化模式（结果仅供参考）。下载 Qlib 数据后可使用完整回测。

### Q: AI 分析报告提示「API Key 认证失败」
检查 `.env` 文件中的 `ANTHROPIC_API_KEY` 是否正确填写，也可在「⚙️ 参数配置」页直接填写并保存。

### Q: AI 报告提示「Claude 不可用，已自动切换到 DeepSeek」
Claude 服务暂时过载（正常现象），已自动降级使用 DeepSeek 生成报告。若不希望降级，稍后重试即可。
若 DeepSeek 也提示失败，检查 `DEEPSEEK_API_KEY` 是否已配置。

### Q: AI 报告生成后在哪里查看
报告自动保存至 `~/Documents/美股交易日记/reports/YYYYMMDD/`，完成后点击弹窗中的「📂 打开目录」可直接打开文件夹。

### Q: PyQt6 启动报错 `xcb` 相关
macOS 上通常不会出现此问题，Linux 上需要安装：
```bash
sudo apt-get install libxcb-cursor0
```

## 8. API Key 配置详解

### 8.1 .env 文件模板

在项目根目录创建 `.env`（参考 `.env.example`）：

```ini
# ── OpenBB 数据接口（均为免费注册）──────────────────────────
# FMP 基本面数据：https://financialmodelingprep.com/register
# 免费：250 次/天，30 年历史财务数据
FMP_API_KEY=

# Finnhub 新闻：https://finnhub.io/register
# 免费：60 次/分钟，无日限额
FINNHUB_API_KEY=

# Alpha Vantage K 线：https://www.alphavantage.co/support/#api-key
# 免费：25 次/天（yfinance 可作为无限量降级源，无需 key）
ALPHA_VANTAGE_API_KEY=

# ── RD-Agent 因子发现（可选，仅使用该功能时需要）────────────
# DeepSeek：https://platform.deepseek.com
# 计费约 ¥0.001/千 Token input，建议预充 ¥50
DEEPSEEK_API_KEY=
CHAT_MODEL=deepseek/deepseek-chat
```

### 8.2 各 Key 申请入口

| Key | 网址 | 免费限额 |
|-----|------|---------|
| FMP | https://financialmodelingprep.com/register | 250 次/天 |
| Finnhub | https://finnhub.io/register | 60 次/分钟 |
| Alpha Vantage | https://www.alphavantage.co/support/#api-key | 25 次/天 |
| DeepSeek | https://platform.deepseek.com | 按 Token 计费 |

### 8.3 在应用内配置

启动后进入「⚙️ 参数配置」→「API Key」标签页，可直接在 UI 中填写并保存（自动写入 `.env`）。

### 8.4 环境预检

```bash
python3 scripts/check_env.py
```

检查内容：PyQt6 / Qlib / OpenBB / Docker / torch 可导入状态，以及 FMP/Finnhub API Key 有效性。
