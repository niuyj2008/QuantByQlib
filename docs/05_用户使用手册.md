# 05 用户使用手册

本手册面向日常使用者，按功能模块介绍各页面的操作方法。

---

## 1. 整体界面说明

应用采用左侧导航栏 + 右侧内容区布局：

```
┌──────────┬─────────────────────────────────────────────┐
│  导航栏  │                 内容区                        │
│          │                                               │
│ 📊 仪表盘 │   当前页面内容                                │
│ 🔍 量化选股│                                               │
│ 📈 选股结果│                                               │
│ 💼 持仓管理│                                               │
│ 🎯 盈利目标│                                               │
│ ⏱  策略回测│                                               │

│ ⚙️  参数配置│                                               │
│ 📋 运行日志│                                               │

```

点击左侧任意菜单项即可切换页面。

## 2. 仪表盘

**入口**：启动后默认显示仪表盘。

### 功能说明

| 区域 | 内容 |
|------|------|
| 顶部 KPI 卡片 | 总持仓市值 / 总盈亏 / 今日涨跌 / 持仓支数 / 目标进度 |
| 左下：最新信号 | 最近一次选股生成的 BUY/SELL 信号预览（最多 10 条）|
| 右下：持仓概况 | 各持仓股票盈亏一览 + 活跃目标进度条 |
| 顶部搜索栏 | 输入股票代码后按 Enter 或点击「🔍 分析」，跳转个股详情 |

### 操作步骤

1. 首次启动时 KPI 均显示「--」，需先录入持仓数据
2. 持仓录入后自动刷新（或点击「刷新价格」手动更新）
3. 运行量化选股后，信号区域自动更新

## 3. 量化选股

**入口**：导航栏「🔍 量化选股」

### 五种策略说明

| 策略 | 适合场景 | 速度 | 支持自定义因子 |
|------|---------|------|--------------|
| 成长股选股（LightGBM）| 日常选股，速度最快 | 快（约 1 分钟）| ✅ |
| 市场自适应（LightGBM+HMM）| 市场转折期 | 中（约 2 分钟）| ✅ |
| 深度学习集成（LSTM）| 趋势市场 | 慢（约 5 分钟）| ❌ |
| 短线获利（GRU）| 短线操作 | 中（约 3 分钟）| ❌ |
| 全市场深度学习（LSTM+Alpha360）| 全面覆盖 | 最慢（约 8 分钟）| ❌ |

> **注意**：Qlib 数据未初始化时，使用备用股票池（100 支大盘股）+ 简化评分，结果仅供参考。

### 已注入自定义因子面板

运行控制区上方显示「🧬 已注入自定义因子」面板，列出当前因子库中通过 IC 验证的自定义因子：

- **鼠标悬停**因子标签，显示 tooltip：因子名（加粗）+ 通俗描述 + 完整 Qlib 表达式
- **右上角数量标注**：当前库中因子数，及"仅 LightGBM 策略使用"提示
- 每次「注入选股策略」完成后，面板**自动刷新**（无需重启或切换页面）
- 应用启动时自动读取上次注入结果并显示

### 操作步骤

1. 选择策略卡片（点击高亮）
2. 若想使用 AI 发现的因子，先前往「因子发现」页完成注入（仅 LightGBM 策略生效）
3. 点击「▶ 开始选股」
4. 等待进度条完成（自动跳转「选股结果」页）

## 4. 选股结果

**入口**：量化选股完成后自动跳转，或导航栏「📈 选股结果」

### 页面布局

```
┌──────────────────────┬───────────────────────────────────┐
│  排名列表             │  个股详情面板（点击左侧行展开）     │
│                      │                                   │
│  # 代码  评分  信号   │  NVDA   $875  ▲+2.3%             │
│  1 NVDA 0.92 买入    │  综合评分：82/100                  │
│  2 AAPL 0.89 买入    │                                   │
│  3 MSFT 0.87 观察    │  [K线图]                           │
│  ...                 │  Alpha158 技术信号                 │
│                      │  基本面（FMP）                     │
│  [导出CSV] [生成信号] │  新闻情绪（Finnhub+VADER）         │
│                      │  [💼 加入持仓] [📊 加入回测]       │
└──────────────────────┴───────────────────────────────────┘
```

### 操作步骤

1. **查看个股详情**：点击左侧表格任意行，右侧面板加载（约 5-10 秒）
2. **加入持仓**：右侧面板点击「💼 加入持仓」，跳转买入对话框
3. **生成信号**：顶部点击「生成信号」，自动跳转「交易信号」页
4. **导出结果**：点击「导出 CSV」保存选股结果

### 个股详情面板内容说明

| 区域 | 数据来源 | 说明 |
|------|---------|------|
| 价格头部 | OpenBB（yfinance）| 当前价、涨跌幅、今日区间 |
| Qlib 评分 | Qlib 模型预测 | 仅从选股结果进入时显示 |
| 综合评分 | 技术 40% + 基本面 40% + 情绪 20% | 0~100 分 |
| Alpha158 技术信号 | Qlib Alpha158 | 动量/均线/量价/波动率 |
| K 线图 | OpenBB Alpha Vantage / yfinance | 含 ATR / OBV / MA |
| 基本面 | OpenBB + FMP | PE / EPS / 营收增速 / 分析师目标价 |
| 新闻情绪 | OpenBB + Finnhub + VADER | 最新 10 条新闻，平均情绪分 |

## 5. 持仓管理

**入口**：导航栏「💼 持仓管理」

### 5.1 录入买入记录

1. 点击右上角「+ 买入」按钮
2. 填写表单：
   - **股票代码**：如 `NVDA`（自动转大写）
   - **股数**：买入数量
   - **成交价**：每股价格（美元）
   - **手续费**：默认 0
   - **交易日期**：默认今日
   - **备注**：可选
3. 点击「确认买入」

系统自动：
- 用均价成本法计算持仓成本
- 通过 OpenBB 获取当前市价计算盈亏

### 5.2 录入卖出记录

1. 点击持仓行右侧「卖出」按钮（或右上角「− 卖出」）
2. 填写卖出股数、成交价、手续费
3. 已实现盈亏自动计算并记录

### 5.3 刷新持仓价格

点击「🔄 刷新价格」，后台通过 OpenBB 批量获取当前市价，更新未实现盈亏。

### 5.4 风险分析

持仓页底部「风险分析」标签：
- **集中度警告**：单股仓位 > 10% 时提示
- **1% 风险法则**：建议单笔最大买入金额 = 总资产 × 1% ÷ 止损比例
- **组合 Beta**：相对市场的系统性风险系数

## 6. 盈利目标

**入口**：导航栏「🎯 盈利目标」

### 6.1 创建目标

1. 顶部选择风险偏好（稳健/平衡/进取）
2. 点击「+ 新建目标」
3. 填写表单：
   - **目标名称**：如「2025 Q1 盈利目标」
   - **期限类型**：月度 / 季度 / 年度
   - **目标收益率**：如 15（代表 15%）
   - **起止日期**
   - **初始资金**：用于计算 1% 风险法则建议仓位
4. 右侧实时预览年化换算

### 6.2 查看进度

目标卡片显示：
- 当前收益率 vs 目标收益率
- 进度条（是否超前/落后）
- 预计到期收益率
- 策略推荐（基于风险偏好）
- 可行性评估（目标年化是否在档位范围内）

> **可行性警告**：若季度 +15% 折算年化 ≈ 60%，超出平衡型档位（12-25%），系统会提示。

## 7. 策略回测

**入口**：导航栏「⏱ 策略回测」

### 操作步骤

1. **选择策略**：与量化选股相同的 5 种策略
2. **设置参数**：
   - 初始资金（默认 $100,000）
   - 开始/结束日期（建议 3 年以上区间）
   - Top-K 持仓数（默认 20）
3. 点击「▶ 开始回测」
4. 等待结果（Qlib 完整回测约 2-5 分钟）

### 结果解读

| 指标 | 说明 | 参考值 |
|------|------|--------|
| 年化收益 | 年化总收益率 | > 20% 为优秀 |
| Sharpe 比率 | 风险调整后收益 | > 1.5 为优秀 |
| 最大回撤 | 历史最大峰谷跌幅 | < -20% 需注意 |
| 年化波动率 | 收益标准差（年化）| < 25% 为稳健 |
| 胜率 | 正收益交易占比 | > 55% 为良好 |
| IC 均值 | 预测分数与实际收益相关性 | > 0.03 为有效 |
| Alpha | 相对 SPY 基准的超额年化收益 | > 0 即有超额 |

净值曲线：紫色为策略净值，灰色为 SPY 基准。

> **降级模式提示**：Qlib 数据未初始化时，显示「等权价格模式」，结果仅供参考，不代表真实回测。

## 8. 交易信号

**入口**：量化选股完成后自动生成；或导航栏「⚡ 交易信号」

### 信号说明

| 信号 | 颜色 | 触发条件 |
|------|------|---------|
| **买入（BUY）** | 绿色 | Qlib 预测分数 Top 10% |
| **持有（HOLD）** | 黄色 | 分数 70-90%，已持仓 |
| **关注（WATCH）** | 蓝色 | 分数 70-90%，未持仓 |
| **卖出（SELL）** | 红色 | 分数 Bottom 20%，已持仓 |

信号强度（弱/中/强）由百分位距离阈值的远近决定。

### 操作说明

1. **筛选信号**：顶部下拉选择「买入」/ 「卖出」/ 「全部」
2. **查看原因**：每条信号附带简短说明（如「Qlib 预测分数 Top 10%，强势买入信号」）
3. **导出 CSV**：点击「📥 导出」保存信号列表

## 9. 因子发现（RD-Agent）

**入口**：导航栏「🤖 因子发现」

> **前提**：Docker Desktop 运行中 + DeepSeek API Key 已配置（参数配置页）+ 已拉取镜像。

### 首次使用

```bash
# Step 1：构建 RD-Agent 镜像（约 2 GB，仅需执行一次）
pip3 install rdagent
rdagent fin_quant

# 或直接拉取预构建镜像
docker pull msrarambler/rd-agent:latest
```

### 因子发现操作步骤

1. 检查页面顶部「Docker 状态」显示「🟢 运行中」
2. 点击「▶ 启动因子发现」
3. 实时查看左侧日志（绿色=正常，红色=错误，黄色=警告）
4. DeepSeek LLM 自动生成 10 个因子候选，容器内完成 IC 验证
5. 发现完成后，右侧表格自动显示（因子名 / 表达式 / IC均值 / Sharpe）
6. 点击「⏹ 停止」可随时中止
7. 完成后点击「📥 导出」保存因子定义为 CSV

### 因子表格说明

| 列 | 说明 |
|----|------|
| 因子名 | 英文下划线命名，如 `momentum_20d` |
| 表达式 | Qlib 表达式，如 `$close/Ref($close,20)-1` |
| IC 均值 | 因子与下期收益的 Spearman 相关系数（> 0.03 为有效）|
| Sharpe | IC 均值 ÷ IC 标准差（IC Information Ratio）|

IC 值颜色标注：绿色（> 0.03）、红色（< -0.01）、灰色（其余）。

### 注入选股策略

发现因子后，点击右侧面板的「✅ 注入选股策略」按钮：

1. 系统对当前会话因子 + 历史因子库进行 **Qlib 实测 IC 验证**（宿主机，30 只蓝筹股，最近 252 个交易日）
2. 进度条实时显示每个因子的验证状态（`[新发现]` 或 `[历史]` 标签）
3. 通过验证（IC ≥ 0.03）的因子写入 `~/.quantbyqlib/valid_factors.json`
4. 旧模型缓存自动清除，**下次运行 LightGBM 选股策略时自动使用新因子**
5. 「量化选股」页的「已注入因子」面板自动刷新

### 已注入因子清单

注入完成后，右侧面板底部显示「已注入因子库」列表：

- 每行显示：`• 因子名：表达式（截断）`
- **鼠标悬停**显示 tooltip：因子名（加粗）+ 通俗中文描述 + 完整 Qlib 表达式
- 该列表**跨会话持久化**：应用关闭再启动后，已注入因子仍然保留

### 因子累积机制

| 机制 | 说明 |
|------|------|
| 合并去重 | 历史库中的因子自动加入本次候选池 |
| 全量重验 | 新因子 + 旧因子统一重跑 IC 验证 |
| 自动淘汰 | 历史中已失效的因子在重验后自然落选 |
| 持续累积 | 每次注入后因子库只增不减（失效除外）|

## 12. 数据说明

### 数据来源汇总

| 功能 | 数据来源 | 是否需要 Key |
|------|---------|------------|
| K 线图 | Alpha Vantage → yfinance（降级）| AV Key（可选，yfinance 无需）|
| Alpha158 技术信号 | Qlib 本地数据 | 需下载 Qlib 数据 |
| 基本面 | FMP | FMP Key（必须）|
| 新闻情绪 | Finnhub | Finnhub Key（必须）|
| 实时价格刷新 | OpenBB（yfinance）| 无需 Key |
| 因子发现 | DeepSeek + Docker | DeepSeek Key（必须）|

### 免费限额注意事项

- **FMP**：250 次/天，10 只股票分析 ≈ 约 30 次 FMP 调用
- **Alpha Vantage**：25 次/天（yfinance 作为无限量降级源）
- **Finnhub**：60 次/分钟，足够日常使用

超出限额时，对应字段显示「暂无数据」，不影响其他功能。

### 无 Mock 原则

本应用严格遵循「零 Mock 数据」原则：
- 任何数据获取失败 → 显示「暂无数据」
- 不使用任何假数据、示例数据、占位数值
- 如看到「暂无数据」，请检查 API Key 和网络连接
