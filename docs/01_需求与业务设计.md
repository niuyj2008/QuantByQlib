# 01 需求与业务设计

## 1. 产品定位

**QuantByQlib** 是面向个人散户投资者的美股量化辅助决策桌面应用。

核心理念：
- **零 Mock 数据**：所有展示数据来自真实数据源，获取失败时显示「暂无数据」
- **复用开源生态**：直接调用 Qlib 0.9.7 内置模型，不重复造轮子
- **散户视角**：操作直观，降低量化门槛，结果可落地执行

目标用户：有一定投资经验、对量化工具感兴趣的美股个人投资者。

---

## 2. 核心功能需求

### 2.1 仪表盘
- 展示持仓总市值、总盈亏（已实现 + 未实现）、持仓支数
- 展示当前活跃盈利目标的完成进度条
- 快速预览最新 BUY/SELL 信号（最多 10 条）
- 搜索框：输入股票代码快速跳转个股分析

### 2.2 量化选股
支持 5 种 Qlib 内置模型策略：

| 策略键 | 名称 | 模型 | 特点 |
|--------|------|------|------|
| `growth_stocks` | 成长股选股 | LightGBM | 速度快，Alpha158 因子 |
| `market_adaptive` | 市场自适应 | LightGBM + HMM | 按市场政体切换 |
| `deep_learning` | 深度学习集成 | LSTM | 捕捉时序模式 |
| `intraday_profit` | 短线获利 | GRU | 短窗口，高换手 |
| `pytorch_full_market` | 全市场深度学习 | LSTM + Alpha360 | 因子最全 |

选股流程：选择策略 → 设置 Top-K → 点击运行 → 实时进度条 → 结果展示。

### 2.3 选股结果
- 左侧：按 Qlib 预测分数降序排列的股票表格
- 右侧：可复用个股详情面板（Alpha158 技术信号 / K 线图 / 基本面 / 情绪）
- 支持导出 CSV、一键加入持仓、生成交易信号

### 2.4 持仓管理
- 录入买入/卖出记录（代码、股数、价格、手续费、日期、备注）
- 均价成本法计算成本价，支持拆股调整
- 持仓概览：总市值 / 总投入 / 已实现 PnL / 未实现 PnL / 综合回报率
- 持仓明细：每支盈亏金额与百分比、今日涨跌
- 风险分析：集中度警告（单股超阈值）、1% 风险法则建议单笔仓位
- OpenBB 实时刷新市价

### 2.5 盈利目标
- 创建目标：名称、期限类型（月度/季度/年度）、目标收益率、起止日期、初始资金
- 进度追踪：当前收益率 vs 目标、是否超前/落后、预计到期收益
- 策略推荐：根据风险偏好（稳健/平衡/进取）推荐策略组合、仓位上限、止损比例
- 可行性分析：年化换算后超出档位范围时给出警告

### 2.6 策略回测
- 参数：策略、日期范围、初始资金、Top-K
- 双模式：Qlib 完整回测（已初始化）/ 等权价格简化回测（降级）
- 结果：年化收益、Sharpe、最大回撤、波动率、胜率、Alpha、IC 统计
- 净值曲线（pyqtgraph）：策略 vs 基准 SPY

### 2.7 交易信号
基于 Qlib 预测分数百分位生成信号：

| 百分位 | 是否持仓 | 信号 |
|--------|---------|------|
| Top 10%（≥90）| 任意 | **BUY** 买入 |
| 70–90% | 已持仓 | **HOLD** 持有 |
| 70–90% | 未持仓 | **WATCH** 关注 |
| Bottom 20%（≤20）| 已持仓 | **SELL** 卖出 |
| 其余 | 任意 | **HOLD** 观望 |

支持按类型筛选、信号强度标注、导出 CSV。

### 2.8 个股分析（可复用面板）
`StockDetailPanel` 组件在三处复用：
1. 选股结果页右侧（带 Qlib 评分）
2. 持仓管理页「查看」按钮
3. 仪表盘搜索框

四个维度并行获取（ThreadPoolExecutor）：
- **技术信号**：Qlib Alpha158 预计算因子
- **K 线图**：OpenBB（alpha_vantage → yfinance 故障转移）+ 本地 ATR/OBV/MA
- **基本面**：OpenBB + FMP（PE / EPS / 营收 / 目标价）
- **情绪**：OpenBB + Finnhub 新闻 + VADER / DistilBERT 双层打分

### 2.9 因子发现（RD-Agent）
- Python docker SDK 启动 `msrarambler/rd-agent:latest` 容器
- 实时流式显示彩色日志（INFO/WARN/ERROR）
- 自动解析因子定义：名称、Qlib 表达式、IC、Sharpe
- 因子列表展示，支持导出 CSV

### 2.10 参数配置
- Qlib 数据路径 + 一键下载 US 数据
- OpenBB API Key（FMP / Finnhub / Alpha Vantage）
- 策略默认参数（Top-K / 止损比例 / 仓位上限）
- RD-Agent 配置（DeepSeek API Key）

## 3. 数据源规划

| 数据类型 | 数据源 | 接入方式 | 免费限额 |
|---------|--------|---------|---------|
| K 线 OHLCV | Alpha Vantage / Yahoo Finance | OpenBB `equity.price.historical` | 25 次/天（AV）/ 无限（yfinance）|
| Alpha158 技术因子 | Qlib 预计算 | `qlib D.features()` | 免费 |
| 基本面 | FMP | OpenBB `equity.fundamental.metrics` | 250 次/天 |
| 新闻标题 | Finnhub | OpenBB `equity.news` | 60 次/分钟 |
| 情绪打分 | VADER（本地）/ DistilBERT（本地） | nltk / transformers | 完全免费 |
| Qlib 训练数据 | Yahoo Finance | Qlib 采集器 | 免费 |
| AI 因子发现 | DeepSeek | RD-Agent Docker | DeepSeek API 计费 |

**多源冗余**：主源失效自动降级（alpha_vantage → yfinance），失败则返回 `None` 并显示「暂无数据」。

---

## 4. 非功能需求

| 维度 | 要求 |
|------|------|
| 响应速度 | 个股分析面板 ≤ 10 秒完成四维并行渲染 |
| UI 流畅 | 所有耗时操作在 QThreadPool 子线程，UI 不阻塞 |
| 可靠性 | 任意数据源失败静默降级，不影响其他维度 |
| 兼容性 | Python 3.9+，macOS 主要，Windows 次要 |
| 安全性 | API Key 存 `.env`，不提交版本控制 |
| Docker | RD-Agent 功能需 Docker Desktop 运行 |
