**`PortfolioManager`**（单例，通过 `get_portfolio_manager()` 获取）

关键方法：
```python
def get_summary() -> dict:
    """
    返回持仓汇总：
    {
        "total_market_value": float,
        "total_cost": float,
        "total_unrealized_pnl": float,
        "total_realized_pnl": float,
        "positions": list[dict],  # 每支股票持仓详情
    }
    """

def get_positions_with_prices() -> list[dict]   # 含实时价格的持仓列表
def get_risk_analysis() -> dict                 # 持仓风险分析（集中度/波动等）
def get_trades(symbol=None) -> list[dict]
```

**持仓计算规则（均价成本法）：**
- 买入：`avg_cost = (现有股数×现有均价 + 新股数×新价格 + 手续费) / 总股数`
- 卖出：已实现 PnL = `(卖出价 - 均价成本) × 卖出股数 - 手续费`
- 未实现 PnL = `(当前市价 - 均价成本) × 持有股数`

### `pnl_calculator.py`
**`PnLCalculator`**：盈亏计算辅助类，封装均价成本法的核心算术，供 `PortfolioManager` 内部调用。

### `risk_analyzer.py`
**`RiskAnalyzer`**：持仓风险分析，产出 `RiskReport` dataclass。

分析维度：
- 持仓集中度（单只股票占比）
- 行业/板块集中度
- 整体波动率估算

## 5. `strategies/` + `screening/` — 量化选股

class StrategyResult:

1. `_qlib_init_check()` — 检查 Qlib 是否初始化
2. `_build_dataset()` — 构建 `DatasetH`（Alpha158 / Alpha360 handler）
3. 模型 fit → predict → `_scores_to_result()`

**`StockScreener.run(strategy_key, topk, progress_cb)`**：
1. `_get_universe()` → `D.list_instruments` 或备用 100 股列表
2. 运行策略 → `StrategyResult`
3. `_batch_get_changes()` → OpenBB 批量获取今日涨跌
4. `_score_to_signal()` → percentile ≥ 90% → BUY，≥ 70% → WATCH，否则 HOLD

## 6. `backtesting/` — 策略回测

### `performance_metrics.py`
**`calc_metrics_from_returns(returns, benchmark_returns=None)`** → `BacktestMetrics`

指标计算：
- `annual_return = (1 + total_return) ** (252/trading_days) - 1`
- `sharpe = mean(r) / std(r) * sqrt(252)`
- `max_drawdown` = 最大峰谷回撤
- `alpha/beta` = 相对基准的超额收益/系统性风险（OLS 回归）

### `backtest_engine.py`
**`BacktestEngine.run(config)`** → `BacktestReport`

双模式：
- **Qlib 全量**：`TopkDropoutStrategy + SignalRecord + PortAnaRecord`
- **简化价格**：等权买入 topk 股票，每日等权重组，基于历史价格计算 NAV

| 模块 | 说明 |
|------|------|
| `price_cache.py` | 历史价格本地缓存，减少重复 OpenBB 请求 |
| `report_generator.py` | 将 `BacktestReport` 格式化为可读 HTML/文本摘要 |

## 9. `strategies/factor_injector.py` — 因子验证与注入

核心函数：

| 函数 | 返回类型 | 说明 |
|------|---------|------|
| `_precheck_expression(expr)` | `(bool, str)` | 语法预检：拦截 Max/Min 非整数窗口、Abs(Ref) 嵌套、一元负号 |
| `validate_factor(expr, universe, threshold_ic)` | `bool` | Qlib 实测截面 Spearman IC，强制 `sequential` 后端 |
| `get_valid_factors(min_ic, progress_cb)` | `list[dict]` | 主流程：预筛 → 合并历史库 → 全量重验证，返回含 name/description 的字典列表 |
| `save_valid_factors(factors)` | `None` | 持久化到 `valid_factors.json`（兼容 `list[dict]` 和 `list[str]`）|
| `load_valid_factors(max_age_hours)` | `list[str]` | 加载表达式列表，TTL 保护（默认 24h）|
| `get_inject_status()` | `dict` | 注入状态摘要，含 `factors` 列表（完整 name/description/expression）|

**合并去重 + 重验证策略**：每次注入时，历史库中的因子自动纳入候选池重验，失效因子自然淘汰，有效因子持续累积。

**macOS 多进程兼容**：`validate_factor()` 强制 `qlib_config.C["joblib_backend"] = "sequential"`，规避 Qt 子线程中 `multiprocessing` spawn 崩溃。

## 10. `workers/` — Qt 后台 Workers

所有 Worker 遵循同一模式：

```python
class XxxSignals(QObject):      # 信号容器（不混入 QRunnable）
    progress  = pyqtSignal(int, str)
    completed = pyqtSignal(object)
    failed    = pyqtSignal(str)

class XxxWorker(QRunnable):
    def __init__(self, ...):
        super().__init__()
        self.signals = XxxSignals()
        self.setAutoDelete(True)

    @pyqtSlot()
    def run(self) -> None:
        try:
            result = do_work()
            self.signals.completed.emit(result)
        except Exception as e:
            self.signals.failed.emit(str(e))
```

启动方式：`QThreadPool.globalInstance().start(worker)`

**现有 Workers：**

| Worker | 说明 |
|--------|------|
| `analysis_worker.py` | 个股四维分析（技术/基本面/K线/情绪），并行 4 线程 |
| `screening_worker.py` | 量化选股：Qlib 模型预测 + OpenBB 涨跌获取 |
| `backtest_worker.py` | 策略回测，Qlib 全量或简化价格双模式 |
| `rdagent_worker.py` | RD-Agent 因子发现，桥接 RDAgentRunner 到 Qt 信号 |
| `factor_inject_worker.py` | 因子注入：IC 验证 + 持久化 + 缓存清除 + EventBus 广播 |
| `price_refresh_worker.py` | 持仓价格批量刷新（OpenBB yfinance）|
| `qlib_downloader.py` | Qlib 美股数据下载 |
| `openbb_test_worker.py` | OpenBB 连通性测试 Worker，配置页用于验证 API Key 是否有效 |
| `yfinance_collector.py` | yFinance 价格数据采集 Worker，作为 OpenBB 的备用数据源 |
