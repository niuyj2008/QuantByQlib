**持仓计算规则（均价成本法）：**
- 买入：`avg_cost = (现有股数×现有均价 + 新股数×新价格 + 手续费) / 总股数`
- 卖出：已实现 PnL = `(卖出价 - 均价成本) × 卖出股数 - 手续费`
- 未实现 PnL = `(当前市价 - 均价成本) × 持有股数`

### `pnl_calculator.py`
**`PnLCalculator`**：盈亏计算辅助类，封装均价成本法的核心算术，供 `PortfolioManager` 内部调用。

### `risk_analyzer.py`
**`RiskAnalyzer`**：持仓风险分析，产出 `RiskReport` dataclass。

分析维度：
- 持仓集中度（单只股票占比）
- 行业/板块集中度
- 整体波动率估算

### `price_chart.py`
**`ChartData`** dataclass：ohlcv DataFrame + 当前价 / 涨跌幅 / ATR14 / OBV 趋势 / 近20日支撑压力 / MA5/MA20/MA60。

**`PriceChart.get_chart_data(ticker, period_days=365)`**：
1. 委托 `data/market_data_client.get_ohlcv_period()`（OpenBB → yfinance 降级链）
2. 本地计算：ATR14 / OBV 趋势 / MA5/MA20/MA60 / 近20日高低点支撑压力

### `fundamental.py`
**`FundamentalData`** dataclass：pe / pb / ps / roe / roa / gross_margin / operating_margin / net_margin / revenue_growth / eps_growth / debt_to_equity / analyst_target / analyst_rating 等。

**`FundamentalAnalyzer.analyze(ticker)`**：
1. 主路径：OpenBB FMP（`get_company_profile` + `get_fundamental_metrics` + `get_earnings_history`）
2. FMP 全部失败时，自动 fallback 到 yfinance `.info`（免费，无需 API Key）
3. **`_sanitize(data, ticker)`** 数据清洗：
   - FMP 部分字段以百分比格式返回（如 ROE=43.25 而非 0.4325）→ 自动 ÷100 换算
   - 换算后仍超出合理范围（ROE/净利率超 ±500%、PE 超 ±1000）→ 丢弃置 None，防止脏数据传入 LLM

### `sentiment.py`
**`SentimentAnalyzer`**：双层情绪分析。

```python
def analyze(ticker: str, use_deep_model: bool = False) -> SentimentData
```

- **VADER**（快速）：NLTK SentimentIntensityAnalyzer，离线，约 1ms/条
- **DistilBERT**（精准）：`mrm8488/distilroberta-finetuned-financial-news-sentiment-analysis`，懒加载约 250MB，约 50ms/条

新闻来源：OpenBB `equity.news`（finnhub → fmp → yfinance 降级）

信号阈值：avg_score > 0.10 → 利好；< -0.10 → 利空；-0.03~0.03 → 中性

### `technical_scorer.py`（新增）
**`TechnicalScorer`**：基于原始 OHLCV 数据计算六维可解释技术评分，与 Alpha158 互补。

**`TechnicalScore`** dataclass：
```python
@dataclass
class TechnicalScore:
    ticker:          str
    ma_trend_score:  Optional[float]   # MA 趋势（30%）
    deviation_score: Optional[float]   # 背离率（20%）
    volume_score:    Optional[float]   # 量能模式（15%）
    macd_score:      Optional[float]   # MACD（15%）
    rsi_score:       Optional[float]   # RSI 动量（10%）
    bband_score:     Optional[float]   # 布林带位置（10%）
    total_score:     Optional[float]   # 0~100 综合分
    signal:          Optional[str]     # 强买/买/观望/卖/强卖
    # 关键原始数值（供 UI 展示和 LLM Prompt 使用）
    ma5/ma10/ma20/ma60, deviation_pct, chase_warning,
    macd_line/signal/hist/cross, rsi6/rsi12/rsi24,
    bband_upper/lower/pct
    available: bool
```

各维度评分逻辑：
| 维度 | 权重 | 评分方法 |
|------|------|---------|
| MA 趋势 | 30% | 多头排列程度（MA5>MA10>MA20>MA60 各层递加）|
| 背离率 | 20% | 价格偏离 MA20；偏离 > 5% 触发 `chase_warning` |
| 量能模式 | 15% | 涨时放量 / 跌时缩量 量价配合度 |
| MACD | 15% | 金叉 / 死叉 / 柱量方向 |
| RSI 动量 | 10% | RSI6/12/24，超买(>70)/超卖(<30)扣分 |
| 布林带 | 10% | %B 值（0=下轨，1=上轨）位置评分 |

**关键方法：**
```python
def score(ticker: str, ohlcv: pd.DataFrame) -> TechnicalScore
def to_dimension_list() -> list[dict]   # 供 UI 雷达图/条形图渲染
```

### `llm_report_generator.py`（新增）
**`LLMReportGenerator`**：LLM 驱动的个股 AI 分析报告生成器。

**`_build_prompt(report: StockReport) -> str`**：
将 `StockReport` 序列化为结构化中文 Prompt，依次包含：当前价格 / 综合评分各维度 / 六维技术评分详情（含 MA 数值、MACD、RSI、布林带）/ 基本面指标 / 情绪与新闻标题。
附加数据诚信约束：严禁虚构数字，缺失数据注明「数据不足」。

**`generate(report) -> str`**：同步生成完整报告（批量场景使用）。

**`generate_stream(report) -> Iterator[str]`**：流式生成（UI 实时显示使用）：

- 非过载错误（401 认证失败等）不降级，直接抛出

报告包含六个章节（System Prompt 约束）：核心结论 / 技术面解读 / 基本面评估 / 情绪与催化剂 / 风险提示 / 作战计划。

### `stock_analyzer.py`
**`StockAnalyzer.analyze(ticker)`**：使用 `ThreadPoolExecutor(max_workers=4)` 并行获取四维数据（30 秒超时），串行追加第五维度，返回 `StockReport`。

**五维数据：**
1. Alpha158 技术信号（Qlib，可选）
2. K 线 OHLCV（`PriceChart` → `market_data_client`）
3. 基本面（`FundamentalAnalyzer`，FMP + yfinance fallback）
4. 新闻情绪（`SentimentAnalyzer`，VADER 默认）
5. 六维技术评分（`TechnicalScorer`，依赖 OHLCV 结果，串行计算）

**Alpha158 无数据时的 fallback**：自动从 OHLCV 推算简化技术信号（`_tech_from_chart()`）。

**综合评分权重（`_calc_overall_score()`，各维度缺失时自动重分配权重）：**
| 维度 | 权重 |
|------|------|
| Alpha158 技术信号 | 25% |
| 六维 OHLCV 技术评分 | 25% |
| 基本面 | 35% |
| 情绪分析 | 15% |

评分等级：≥75=强势买入，≥60=买入，≥45=持有，≥30=观望，<30=卖出。

## 8. `backtesting/` — 策略回测

### `performance_metrics.py`
**`calc_metrics_from_returns(returns, benchmark_returns=None)`** → `BacktestMetrics`

指标计算：
- `annual_return = (1 + total_return) ** (252/trading_days) - 1`
- `sharpe = mean(r) / std(r) * sqrt(252)`
- `max_drawdown` = 最大峰谷回撤
- `alpha/beta` = 相对基准的超额收益/系统性风险（OLS 回归）

### `backtest_engine.py`
**`BacktestEngine.run(config)`** → `BacktestReport`

双模式：
- **Qlib 全量**：`TopkDropoutStrategy + SignalRecord + PortAnaRecord`
- **简化价格**：等权买入 topk 股票，每日等权重组，基于历史价格计算 NAV

### `signal_validator.py`（新增）
**`SignalValidator`**：历史信号胜率验证，回溯已导出的买入信号实际收益。

**核心方法：**
```python
def validate(lookback_days: int = 60) -> list[dict]
# 扫描 signals/ 目录下最近 N 天的信号 CSV 文件
# 对每条买入信号，用 yfinance 获取 T+5 / T+20 实际价格
# 计算实际收益率，判断是否「胜」（收益 > 0）
# 返回：[{ticker, signal_date, strategy, ret_t5, ret_t20, win_t5, win_t20}]
```

回测结果汇总指标：胜率（T+5 / T+20）、平均收益、有效信号总数。

## 11. `strategies/factor_injector.py` — 因子验证与注入

核心函数：

| 函数 | 返回类型 | 说明 |
|------|---------|------|
| `_precheck_expression(expr)` | `(bool, str)` | 语法预检：拦截 Max/Min 非整数窗口、Abs(Ref) 嵌套、一元负号 |
| `validate_factor(expr, universe, threshold_ic)` | `bool` | Qlib 实测截面 Spearman IC，强制 `sequential` 后端 |
| `get_valid_factors(min_ic, progress_cb)` | `list[dict]` | 主流程：预筛 → 合并历史库 → 全量重验证，返回含 name/description 的字典列表 |
| `save_valid_factors(factors)` | `None` | 持久化到 `valid_factors.json`（兼容 `list[dict]` 和 `list[str]`）|
| `load_valid_factors(max_age_hours)` | `list[str]` | 加载表达式列表，TTL 保护（默认 24h）|

**合并去重 + 重验证策略**：每次注入时，历史库中的因子自动纳入候选池重验，失效因子自然淘汰，有效因子持续累积。

## 12. `workers/` — Qt 后台 Workers

所有 Worker 遵循同一模式：

```python
class XxxSignals(QObject):      # 信号容器（不混入 QRunnable）
    progress  = pyqtSignal(int, str)
    completed = pyqtSignal(object)
    failed    = pyqtSignal(str)

class XxxWorker(QRunnable):
    def __init__(self, ...):
        super().__init__()
        self.signals = XxxSignals()
        self.setAutoDelete(True)

    @pyqtSlot()
    def run(self) -> None:
        try:
            result = do_work()
            self.signals.completed.emit(result)
        except Exception as e:
            self.signals.failed.emit(str(e))
```

启动方式：`QThreadPool.globalInstance().start(worker)`

**现有 Workers：**

| Worker | 说明 |
